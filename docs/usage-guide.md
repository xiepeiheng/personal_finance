# 个人财务系统使用指南

基于你 2025 年账单数据的分析结果，这份指南帮助你快速上手。

---

## 一、账本设计（Ledger）

账本有两种类型，你可以根据自己的习惯创建。

### 时间维度账本（`ledger_type: time`）

以时间周期划分的账本，用于查看某段时间内的财务状况：

```
2025年总账           ← 全年汇总（必建）
2025年1月           ← 你的月周期是 15 日～下月 14 日
2025年2月
...
2025年12月
```

**创建建议：**
- 先建一个 `2025年总账` 作为全年汇总
- 每月月中旬创建一个新的月度账本（如 6 月 15 日建 `2025年6月`）
- 后续年份类似，建 `2026年总账` 并按月创建子账本

### 分类维度账本（`ledger_type: category`）

按消费类别划分的账本，用于归类同类开销：

```
日常开销        ← 三餐、通勤、日用品
固定支出        ← 房租、电费、话费
娱乐            ← 电影、游戏、咖啡、酒吧
人际关系        ← 杨楚珩相关、朋友聚餐、礼金
工作相关        ← 待报销、未报销
家庭            ← 父母、亲戚
大额消费/旅行    ← 临时性大额支出（用后关闭）
```

## 二、分类设计（Category）

分类是系统的最小粒度统计单位。每个分类同时关联一个时间账本和一个（或零个）分类账本。

| 分类名称 | 关联时间账本 | 关联分类账本 | 预算参考 | 说明 |

| 三餐 | 2025年总账 + 当前月账本 | 日常开销 | ¥3,000 | 正餐、外卖 |
| 饮料零食 | 2025年总账 + 当前月账本 | 日常开销 | ¥500 | 奶茶、咖啡、零食 |
| 通勤 | 2025年总账 + 当前月账本 | 日常开销 | ¥500 | 滴滴、公交、共享单车 |
| 房租 | 2025年总账 + 当前月账本 | 固定支出 | ¥3,000 | |
| 话费会员 | 2025年总账 + 当前月账本 | 固定支出 | ¥100 | 话费、网易云/B站/Pixiv 等订阅 |
| 医疗 | 2025年总账 + 当前月账本 | 日常开销 | ¥200 | 牙科、药品 |
| 娱乐 | 2025年总账 + 当前月账本 | 娱乐 | ¥500 | 电影、游戏充值、酒吧 |

### 临时分类（用后关闭 `is_complete = true`）

| 分类名称 | 说明 |
|---------|------|
| 跨年旅行 | 旅行结束后标记完成 |
| 车辆维护 | 按需创建，修完即止 |
| 某某项目 | 任何一次性活动支出 |

### 收入分类

| 分类名称 | 说明 |
|---------|------|
| 工资收入 | 每月工资 |
| 提成收入 | 杨楚珩发放的提成 |
| 礼金红包 | 生日/节日红包 |
| 报销 | 公司报销款 |

## 三、记账实操示例

### 示例 1：一次日常消费

场景：2025-06-17，美团外卖晚餐 ¥41.60，滴滴通勤 ¥13.00

**操作步骤：**
1. 进入"交易记录"页面，点击"新增交易"
2. 填写：

| 字段 | 值 |
|------|-----|
| 分类 | 三餐 |
| 交易日期 | 2025-06-17 |
| 交易对象 | 美团外卖 |
| 金额 | -41.60 |
| 交易渠道 | 微信支付 |

3. 提交 → 系统自动将 -41.60 累计到：
   - 分类 `三餐` 的 `actual_amount`
   - 时间账本 `2025年6月` 的 `balance`
   - 分类账本 `日常开销` 的 `balance`
   - 时间账本 `2025年总账` 的 `balance`

### 示例 2：待报销的工作支出

场景：一笔 ¥1,647.56 的腾讯云续费需要走报销

**操作步骤：**
1. 新建交易，分类选 `待报销`，金额 -1647.56
2. 报销到账后：编辑该交易，修改分类从 `待报销` 改为实际用途（如 `技术支出`）
3. 如果报销金额 ≠ 支出金额，可以新建一笔收入交易作为"报销收入"

### 示例 3：杨楚珩相关的支出

场景：和杨楚珩出去玩，你方支出 ¥282.00（酒吧）

**操作步骤：**
1. 分类选 `杨楚珩`
2. 金额 -282.00
3. 备注可以写具体活动
4. 系统自动在 `人际关系` 分类账本中累计，月底可以查看在该账本下花了多少

## 四、常用查询

| 想查什么 | 操作方法 |
|---------|---------|
| 本月总共花了多少钱 | 首页 Dashboard → 查看"本月支出" |
| 本月各分类花了多少 | 首页 → 分类支出排行 |
| 和杨楚珩今年总共花了多少 | 分类账本 `人际关系` 的 `balance` |
| 某个月份的详细记录 | 按月筛选 Transactions 页 |
| 某笔历史交易 | Transactions 页 → 搜索交易对象名称 |
| 预算还剩多少 | 查看分类的 `budget - actual_amount` |
| 哪些分类还没关闭 | 按状态筛选 ≠ 已完成 |

## 五、关键要点

1. **金额正负规则**：正数 = 收入，负数 = 支出
2. **月末核对**：创建新账本时，检查上个月的账是否都记完了
3. **报销处理**：先记支出，到账后调整分类
4. **分类复用**：同一个分类可以同时关联时间账本和分类账本，一个交易同时计入两个账本
5. **预算提醒**：首页 Dashboard 如果看到某个分类的金额变成红色，说明超预算了

## 六、推荐初始设置流程

```
第一步：创建时间账本
  ├── 2025年总账（time）
  └── 当前月账本（time 如 2025年6月）

第二步：创建分类账本
  ├── 日常开销（category）
  ├── 固定支出（category）
  ├── 娱乐（category）
  ├── 人际关系（category）
  └── 工作相关（category）

第三步：创建分类并关联账本
  按上面的分类表逐一创建，每个分类关联一个时间账本和一个分类账本

第四步：录入第一条交易
  熟悉操作流程后，开始批量录入历史数据

第五步：设置预算
  为每个常规分类设置合理的 budget 值
```
